# AutoClash

AutoClash 是一款基于 Android 平台的自动化代理组切换工具。它可以根据你当前所处的网络环境（如连接的 WiFi 或者使用的网络运营商），自动调用 Mihomo (Clash Meta) 的 API 来切换相应的策略组节点。

## ✨ 核心特性

- **基于环境的智能切换**：
  - **WLAN (WiFi)**：根据当前连接的 WiFi SSID 名称自动切换策略组。
  - **运营商 (ISP)**：根据当前使用的网络运营商（如中国电信、中国联通、中国移动）自动切换策略组。
- **Mihomo (Clash Meta) 集成**：通过 RESTful API 直接与 Mihomo 核心通信，获取策略组详情并安全地执行代理切换。
- **灵活的规则引擎**：支持多条自动化规则、设置规则优先级，并具备“规则取反”功能（例如：当**不**连接某个 WiFi 时执行某动作）。
- **后台常驻服务**：提供稳定的前台服务（Foreground Service）监听网络变化。支持一键申请忽略电池优化，确保服务在后台稳定运行。
- **现代美观的 UI**：采用最新的 Jetpack Compose 和 Material 3 设计构建。

## 🚀 快速开始

### 使用步骤

1. **配置 API**：
   - 打开 AutoClash 应用的设置页面。
   - 输入 Mihomo 的 API 地址（例如：`http://127.0.0.1:9090`）。
   - 如果你的 Mihomo 核心配置了 Secret，也请在此填入。
   - 点击“测试连接”，确保 AutoClash 可以成功连接到 Mihomo 核心。
2. **添加自动化规则**：
   - 进入“策略组管理”，在这里你可以看到所有的 Selector 类型策略组。
   - 为特定的策略组添加自动化规则，例如：如果 WiFi 等于 `MyHome`，则将策略组 `Proxy` 切换到 `DIRECT`。
3. **启动服务**：
   - 在设置页面中开启“后台服务”。
   - 为了确保切换及时生效且不会被系统杀后台，**强烈建议**在设置中点击“请求忽略电池优化”。
4. **授予权限**：
   - 根据系统版本，如果你使用 WiFi 判定规则，应用可能需要**位置权限**来获取当前连接的 WiFi 名称（SSID）。
